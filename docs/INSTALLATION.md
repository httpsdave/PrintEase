# PrintEase Installation and Distribution Guide

## 1. Prerequisites

- Windows 10 or Windows 11
- .NET SDK 8.0+ (for building)
- Inno Setup 6+ (for creating installer)
- Optional: code-signing certificate for trusted installs

## 2. Build release output

From the workspace root:

```powershell
dotnet publish .\PrintEase.App\PrintEase.App.csproj -c Release -r win-x64 --self-contained true /p:PublishSingleFile=true
```

Publish output:

- `PrintEase.App\bin\Release\net8.0-windows\win-x64\publish\PrintEase.App.exe`

## 3. Terms and Conditions in installer

- License terms are stored in `LICENSE-EULA.txt`.
- Inno Setup displays these terms during installation through `LicenseFile`.

## 4. Logo and branding

- Replace `assets/logo.svg` with your final brand logo.
- Export an ICO file for installer/app icon, for example: `assets/logo.ico`.
- Update the installer script with your icon path.

## 5. Build installer (Inno Setup)

1. Open Inno Setup Compiler.
2. Open `installer/PrintEase.iss`.
3. Update:
   - `MyAppPublisher`
   - `MyAppURL`
   - icon fields (if used)
4. Click Build.

Generated installer output:

- `installer\Output\PrintEase-Setup.exe`

Alternative scripted build:

```powershell
.\scripts\build-installer.ps1
```

## 6. Download location options

Use at least one of these:

1. GitHub Releases:
   - Create a release tag.
   - Upload `PrintEase-Setup.exe`.
   - Share the release URL.
2. Your website:
   - Host `PrintEase-Setup.exe` in a downloads page.
3. Microsoft Store (optional later):
   - Package via MSIX process.

## 7. Professional release checklist

- Add real company name and support email.
- Replace placeholder EULA text with legal-approved terms.
- Add logo/icon assets.
- Sign executable and installer.
- Verify install/uninstall on a clean Windows machine.
- Smoke test with both USB and Wi-Fi printers.

## 8. Known admin requirement

Restarting the print spooler generally requires Administrator rights.

## 9. Local app data generated by PrintEase

PrintEase stores local state here:

- `%LOCALAPPDATA%\\PrintEase\\profiles.json` for per-printer profiles
- `%LOCALAPPDATA%\\PrintEase\\presets.json` for custom presets

Diagnostics storage behavior:

- Diagnostics and snapshots are written to `PrintEaseData\\logs` on a non-Windows drive only (for example `D:\\PrintEaseData\\logs`).
- If no non-Windows drive is available, diagnostics export is blocked until one is available.

## 10. Presets and auto-recovery usage

- Use the Preset controls in the app to save and reapply common print settings.
- Built-in presets are `Balanced`, `FastDraft`, and `PhotoHigh`.
- Auto Recover Printer runs:
   1. connection pre-check
   2. spooler restart
   3. optional queue purge confirmation
   4. post-check with status update

## 12. Print size and file support

- Paper sizes available in-app: `A4`, `Letter`, `Legal`, `8x13`, `A3`, `A5`, and `Custom` inches.
- You can attach files for print jobs (PDF, image formats, Word docs, text files).
- Image files show inline preview; other file types show metadata preview and are sent using registered app print handlers.

## 11. GitHub release automation

- Workflow file: `.github/workflows/release.yml`
- Trigger: push a semantic tag like `v1.0.3`
- Optional signing secrets:
   - `SIGN_CERT_BASE64`
   - `SIGN_CERT_PASSWORD`
